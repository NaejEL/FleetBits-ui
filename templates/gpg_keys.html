KEY MANAGEMENT

What is a GPG Key?

A GPG key is a cryptographic key used to digitally sign Debian package metadata. When you publish packages to Aptly, the repository is signed with your GPG key so that devices can verify the authenticity of packages before installation.

How to Use Keys

  • Generate new: Create a new key for this FleetBits instance (recommended for new deployments)
  • Import existing: Use an existing organizational GPG key if you have one
  • Delete unused: Remove old or revoked keys

Best Practices

  • Use RSA 4096-bit or EdDSA for new keys (better security)
  • Use an organizational email address (e.g., release@your-org.com)
  • Keep the private key in your Ansible vault or secure storage
  • Rotate keys annually or after personnel changes
  • Back up exported keys in a secure location
